But why should I care about Ocaml?

Running a speed test on the fibonacci sequence, one of the classic recursion problems used to demonstrate the speed and effectiveness of a language, as we increment the value of our fibonacci index, we see that the difference in time between our Python and C compile time differs greatly, with C running faster due to it being more optimized towards dealing with calculations due to its static-typed nature to Python's dynamically typed nature. However, Ocaml, being a high-level language manages to keep up close with C compile time around a second by also being static-typed. This allows whoever is using Ocaml to write simple, efficient code with fast processes. Ocaml has many libraries it shares with other dynamically-typed languages, which is why it was developed into one of the modern langauges Rust, and is technically used to this day to create an array of products such as Discord.
